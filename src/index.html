Reactive Principles

  • Streams
  • Immutability
  • Reactivity

A stream is a sequence of events over time

A stream can emit three kinds of events:
    values
    an error
    a complete notification

After an error or a complete notification, nothing else is emitted by the stream.

Rx Operators never modifies the input stream. Instead it return a new streams. This principle is called Immutability.

Reactive programming is programing with streams. In reactive programming we react to event/data and stream updates. This is the reactive principle.

A listener can receive three kinds of events [ value, error, complete ]. So, it can consume the values, handle an error and act upon completion of the subscribed stream.
